Build your first Hedera dApp with modern React, TypeScript, and Mirror Node integration.

## Prerequisites

Before you begin, ensure you have:

- **Node.js 18+** installed on your system
- **MetaMask** or **HashPack** wallet extension (ECDSA account recommended for HashPack)
- Basic familiarity with React and TypeScript

## Installation

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/your-org/hedera-dx-starter.git
    cd hedera-dx-starter/frontend
    ```
  </Step>

  <Step title="Install dependencies">
    ```bash
    npm install
    ```
  </Step>

  <Step title="Configure environment (optional)">
    Create a `.env.local` file in the frontend directory:

    ```env
    NEXT_PUBLIC_NETWORK=testnet
    NEXT_PUBLIC_MIRROR_NODE_URL=https://testnet.mirrornode.hedera.com
    ```

    <Note>
      These are the default values. You only need to create this file if you want to customize the network or Mirror Node URL.
    </Note>
  </Step>

  <Step title="Start the development server">
    ```bash
    npm run dev
    ```

    Open [http://localhost:3000](http://localhost:3000) in your browser.
  </Step>
</Steps>

## Connect Your Wallet

The starter kit supports MetaMask and HashPack via WalletConnect. Here's how to connect:

```tsx
import { useWallet } from '@/contexts/WalletContext';

export function MyComponent() {
  const { wallet, setWallet } = useWallet();

  const handleConnect = async () => {
    // For MetaMask
    const accounts = await window.ethereum.request({
      method: 'eth_requestAccounts',
    });

    setWallet({
      accountId: accounts[0],
      balance: '0.00',
      isConnected: true,
      provider: window.ethereum,
      walletType: 'metamask'
    });
  };

  return wallet.isConnected ? (
    <p>Connected: {wallet.accountId}</p>
  ) : (
    <button onClick={handleConnect}>Connect Wallet</button>
  );
}
```

## Fetch Account Data

Once connected, use the Mirror Node service to fetch account information:

```tsx
import { MirrorNodeService } from '@/services/MirrorNodeService';

// Get the singleton instance
const mirror = MirrorNodeService.getInstance();

// Fetch account info
const accountInfo = await mirror.getAccountInfo('0.0.1234');
console.log(`Balance: ${mirror.formatHbar(accountInfo.balance.balance)} HBAR`);

// Fetch token balances with metadata
const tokens = await mirror.getAccountTokenBalancesWithInfo('0.0.1234');
tokens.forEach(token => {
  console.log(`${token.symbol}: ${mirror.formatTokenAmount(token.balance, token.decimals)}`);
});

// Fetch recent transactions
const transactions = await mirror.getAccountTransactions('0.0.1234', 10);
transactions.forEach(tx => {
  console.log(`${tx.name} - ${tx.result}`);
});
```

<Tip>
  The Mirror Node service automatically handles EVM addresses (0x...) and converts them to Hedera account IDs (0.0.xxxxx).
</Tip>

## Troubleshooting

<AccordionGroup>
  <Accordion title="MetaMask not connecting">
    Ensure MetaMask is installed and the Hedera Testnet network is added. The starter kit automatically prompts to add the network on first connection.

    If issues persist, manually add the network:
    - **Network Name**: Hedera Testnet
    - **RPC URL**: `https://testnet.hashio.io/api`
    - **Chain ID**: 296 (0x128)
    - **Currency Symbol**: HBAR
  </Accordion>

  <Accordion title="HashPack connection failing">
    Make sure you're using an **ECDSA account** in HashPack, not an ED25519 account. WalletConnect requires ECDSA for compatibility.

    To create an ECDSA account in HashPack:
    1. Open HashPack extension
    2. Go to Settings → Accounts
    3. Create new account with ECDSA key type
  </Accordion>

  <Accordion title="Mirror Node API errors">
    If you see 404 errors when fetching account data:
    - Verify the account exists on testnet using [HashScan](https://hashscan.io/testnet)
    - Ensure you're using the correct account ID format (0.0.xxxxx)
    - Check that `NEXT_PUBLIC_MIRROR_NODE_URL` points to the correct network
  </Accordion>

  <Accordion title="Build or runtime errors">
    Common fixes:
    - Clear Next.js cache: `rm -rf .next`
    - Reinstall dependencies: `rm -rf node_modules package-lock.json && npm install`
    - Ensure Node.js version is 18 or higher: `node --version`
  </Accordion>
</AccordionGroup>
